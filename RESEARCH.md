# Ресёрч: Threads для лидогенерации в нише рекламы

> Спринт 1, дата: 2026-06-26. Источник выводов — веб-ресёрч (RU + EN гайды по алгоритму Threads 2026, документация Threads API). Ссылки внизу.

## Задача и позиционирование

**Цель:** автопостинг в Threads → лиды на настройку рекламы.
**Основные офферы:** Google Ads и Meta Ads. Дополнительно: VK Реклама, Яндекс Директ, сайты, лендинги, таплинки, боты, вайбкодинг.
**ЦА (ключевое):** русскоязычные предприниматели/малый бизнес **за рубежом** (эмигранты). Им нужны именно Google/Meta Ads (работают там, где они живут), а не VK/Яндекс.
**Язык:** русский (основной), английский — изредка под гугл/мета тематику.

## 1. Как работает алгоритм Threads (2026)

Лента **For You** — AI-ранжирование: пользователь видит много контента от тех, на кого НЕ подписан. Это шанс залететь в охваты без большой базы.

**Сигналы ранжирования (по убыванию важности):**

1. **Скорость вовлечённости (engagement velocity).** Главный сигнал — **ответы в первые ~30 минут**. 20 ответов за 30 минут раскручивают пост сильнее, чем 50 ответов за сутки.
2. **Качество дискуссии.** Ценятся содержательные ответы и **то, что автор сам отвечает в своей ветке** (re-engage). Алгоритм 2026 поднимает наверх не первые, а самые ценные комментарии.
3. **Формат.** Text-only посты заходят ХУЖЕ всего. **Текст + картинка = максимальный охват.**
4. **Завершение постом-вопросом** / приглашением к спору → больше ответов → больше охвата.
5. **Ссылки НЕ режут охват** в 2026 (в отличие от X), если вокруг поста есть живое обсуждение.
6. В 2025 был ребаланс — чуть больше веса аккаунтам, на которые уже подписан.

**Практические следствия для нас:**

- Каждый пост заканчивается вопросом/затравкой к дискуссии.
- К каждому посту по возможности — простой визуал (скрин кабинета, инфографика с цифрой).
- Первые 30 минут после публикации — самое важное: отвечать на комментарии, можно «затравить» вопросом в первом комментарии.
- Каденс: **2-5 постов/день**, минимум 1 короткий разговорный + 1 визуальный.
- 80% оригинального контента / 20% адаптированного.

## 2. Что заходит в нише (форматы и хуки)

Структура вирусного поста: **хук (проблема/интрига) → тело (суть, шаги) → CTA (вывод + призыв к диалогу)**.

**Рабочие форматы:**
- Кейсы с конкретными цифрами (бюджет → лиды → CPL/ROAS).
- Ошибки «на чём сливают бюджет».
- Мини-гайды в 2-4 шага.
- Разрушение мифов про рекламу.
- Боль эмигранта-предпринимателя (запуск рекламы в новой стране, оплата, локализация).
- Спорные мнения + приглашение поспорить.

**Хуки** (первая строка решает всё) — см. `src/content/hooks.js`: цифра-результат, дорогая ошибка, спорное мнение, прямое обращение, список, вопрос-затравка, развенчание мифа, до/после, признание.

**CTA:** ~70% разговорных (вопрос аудитории — гонит комментарии и охват), ~30% прямых лид-CTA (в директ). Жёсткая продажа убивает охват.

Эти выводы зашиты в контент-движок (`src/content/`) и в системный промпт генератора (`src/llm.js`).

## 3. Threads API (официальный, Meta for Developers)

Выбранный способ постинга — официальный API (легально, без риска бана).

**Что нужно для доступа:**
- Профессиональный (business) аккаунт Threads.
- Приложение в Meta for Developers + access token (OAuth, scope на публикацию).

**Лимиты и возможности:**
- Бесплатно. Лимит: **250 постов / 24ч** на профиль. Ответы (до 1000/день) НЕ считаются в эти 250.
- Типы контента: текст (≤500 симв.), одно фото, одно видео, карусель (до 10 элементов, считается за 1 пост).
- **Двухшаговая публикация:** `POST /{user-id}/threads` (создать контейнер) → подождать **≥30 сек** → `POST /{user-id}/threads_publish` (опубликовать по id контейнера).
- Картинка: JPEG/PNG, ≤8 МБ, ширина 320-1440 px. Видео: MP4/MOV, ≤5 мин, ≤1 ГБ.
- Можно автоматизировать постинг по расписанию.

**Это будет основой постера в спринте 2** (сейчас не реализовано — генератор пишет очередь в `data/queue.json`).

## 4. Что уже сделано (спринт 1) и что дальше

**Сделано:**
- Ресёрч алгоритма + API (этот документ).
- Контент-движок: банк рубрик/хуков/архетипов/CTA (`src/content/`) + генератор постов через Gemini API (`@google/genai`, по умолчанию `gemini-3.5-flash`) с заточкой под алгоритм (`src/generate.js`, `src/llm.js`). Выдаёт очередь постов с расписанием в `data/queue.json`.

**Дальше (спринт 2+):**
1. Постер на Threads API (двухшаговая публикация + загрузка визуала + соблюдение лимитов).
2. Авто-расписание (cron) и публикация из очереди.
3. Авто-визуал к постам (генерация картинок/инфографики).
4. Трекинг: какие посты дали комментарии/лиды → дообучение банка хуков.
5. Подсказки по «прогреву» первых 30 минут (первый комментарий-затравка).
