# treads — авто-постинг в Threads для лидогенерации

Генерирует и (в будущем) публикует посты в Threads под нишу **настройки рекламы
(Google Ads / Meta Ads)** для **русскоязычного бизнеса за рубежом**.

Контент заточен под алгоритм Threads (ранняя вовлечённость, посты-вопросы,
текст+картинка) — подробности в [`RESEARCH.md`](./RESEARCH.md).
Генерация постов — через **Gemini** (`@google/genai`, по умолчанию `gemini-3.5-flash`).

## Структура

```
RESEARCH.md            ← ресёрч: алгоритм Threads + API + форматы ниши
src/
  content/
    pillars.js         ← рубрики (темы) + офферы
    hooks.js           ← формулы хуков
    archetypes.js      ← структуры поста (хук→тело→CTA)
    cta.js             ← призывы (разговорные / прямые)
  llm.js               ← генерация поста через Gemini API
  generate.js          ← движок: брифы → очередь постов
data/
  queue.json           ← результат: очередь постов с расписанием (создаётся при запуске)
```

## Запуск

Офлайн (скелеты постов, без ключа и без установки зависимостей):

```bash
node src/generate.js --count 5 --no-ai
```

Полная генерация через Gemini (готовые посты):

```bash
npm install
$env:GEMINI_API_KEY="..."          # PowerShell (cmd: set GEMINI_API_KEY=...)
node src/generate.js --count 7
```

Параметры:
- `--count N` — сколько постов сгенерировать (по умолчанию 5).
- `--no-ai` — режим скелетов без обращения к API.
- `GEMINI_MODEL` — переопределить модель (по умолчанию `gemini-3.5-flash`).

Результат — `data/queue.json`: посты с текстом, флагом `needs_image`, идеей
визуала, хэштегами и временем публикации (`scheduled_at`).
